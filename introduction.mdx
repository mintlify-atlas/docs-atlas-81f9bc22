## Overview

This framework provides a complete **Behavior-Driven Development (BDD)** solution for automated API testing. Built on top of [REST-Assured](https://rest-assured.io/) and [Cucumber](https://cucumber.io/), it enables teams to write human-readable test scenarios in Gherkin that directly validate REST API behavior.

The framework targets a Google Maps-style Places API (hosted at `https://rahulshettyacademy.com`) and demonstrates a full CRUD testing workflow: creating a place, reading it back, and deleting it — all verified end-to-end.

## What is being tested

The framework validates three API endpoints of a Places API:

| Endpoint | Method | Purpose |
|---|---|---|
| `/maps/api/place/add/json` | POST | Create a new place |
| `/maps/api/place/get/json` | GET | Retrieve a place by ID |
| `/maps/api/place/delete/json` | POST | Delete a place by ID |

All requests are authenticated with an API key (`key=qaclick123`) passed as a query parameter on every call.
